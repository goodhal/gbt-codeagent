# GB/T 34946-2017 C#语言源代码漏洞测试规范

C# 语言源代码安全漏洞检测规则集，基于 GB/T 34946-2017 标准

**标准**: GB/T 34946-2017
**语言**: C#
**规则数量**: 42


---

## 6.2.1 行为问题

### 问题分类GB/T34946-6.2.1.1 不可控的内存分配 🟠

**严重级别**: HIGH | **CWE**: [CWE-789](https://cwe.mitre.org/data/definitions/789.html)


在未检查用户输入大小的情况下分配内存，可能导致内存耗尽或拒绝服务


**审计要点**:

检查代码中是否存在未验证用户输入大小就直接进行内存分配的操作，如 new byte[]、ArrayList 扩容等


**修复建议**:

在内存分配前验证用户输入的大小，设置合理的上限；使用安全的数据结构如 List<T> 并设置容量上限

## 6.2.2 路径错误

### 问题分类GB/T34946-6.2.2.1 不可信的搜索路径 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-426](https://cwe.mitre.org/data/definitions/426.html)


代码使用用户可控制的路径搜索资源，可能导致路径遍历攻击


**审计要点**:

检查是否使用用户输入构建文件路径；检查 Path.Combine、LoadLibrary 等函数的输入是否经过验证


**修复建议**:

避免使用用户输入构建路径；使用白名单验证路径；使用 Path.GetFullPath 并检查结果是否在预期目录内

## 6.2.3 数据处理

### 问题分类GB/T34946-6.2.3.1 相对路径遍历 🟠

**严重级别**: HIGH | **CWE**: [CWE-22](https://cwe.mitre.org/data/definitions/22.html)


使用用户控制的相对路径访问文件，可能导致目录遍历攻击


**审计要点**:

检查文件操作函数（File.ReadAllText、StreamReader 等）是否使用用户输入的相对路径；是否存在 ../ 路径穿越


**修复建议**:

使用 Path.GetFullPath 将路径规范化；验证路径在允许的目录内；使用白名单验证文件名称

### 问题分类GB/T34946-6.2.3.2 绝对路径遍历 🟠

**严重级别**: HIGH | **CWE**: [CWE-36](https://cwe.mitre.org/data/definitions/36.html)


使用用户控制的绝对路径访问文件，可能导致访问任意文件


**审计要点**:

检查是否直接使用用户输入作为文件路径；Path.GetFullPath 是否正确处理路径遍历序列


**修复建议**:

对用户输入的路径进行规范化并验证是否在允许范围内；避免直接使用用户输入访问文件系统

### 问题分类GB/T34946-6.2.3.3 命令注入 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-78](https://cwe.mitre.org/data/definitions/78.html)


使用用户输入构造系统命令，可能导致远程命令执行


**审计要点**:

检查 Process.Start、ProcessInfo、ShellExecute 等调用是否包含用户输入；是否存在字符串拼接构建命令


**修复建议**:

避免使用用户输入构造命令；使用参数化方式传递命令参数；使用白名单验证输入；优先使用 API 而非命令行

### 问题分类GB/T34946-6.2.3.4 SQL注入 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-89](https://cwe.mitre.org/data/definitions/89.html)


使用字符串拼接构建 SQL 语句，攻击者可通过输入改变语句逻辑


**审计要点**:

检查是否存在字符串拼接或字符串格式化构建 SQL；检查是否使用参数化查询；检查 ORMCriteria、LINQ 是否安全


**修复建议**:

使用参数化查询（SqlParameter、Entity SQL 参数）；使用 ORM 的参数化方法；避免动态 SQL 构建

### 问题分类GB/T34946-6.2.3.5 代码注入 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-94](https://cwe.mitre.org/data/definitions/94.html)


使用用户输入动态执行代码，可能导致远程代码执行


**审计要点**:

检查是否存在 Eval、Reflection、Dynamic、ScriptEngine 等动态代码执行；检查是否使用用户输入构造 lambda 或表达式树


**修复建议**:

避免使用动态代码执行；使用安全的数据处理方式；如必须使用，限制执行环境和输入白名单

### 问题分类GB/T34946-6.2.3.6 进程控制 🟠

**严重级别**: HIGH | **CWE**: [CWE-114](https://cwe.mitre.org/data/definitions/114.html)


使用用户输入控制进程行为，可能导致权限提升或命令执行


**审计要点**:

检查 ProcessStartInfo 的属性（FileName、Arguments、WorkingDirectory）是否来自用户输入且未经验证


**修复建议**:

验证 Process 属性的合法性；使用白名单限制可执行文件；避免将用户输入直接用于进程参数

### 问题分类GB/T34946-6.2.3.7 信息通过错误消息泄露 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-209](https://cwe.mitre.org/data/definitions/209.html)


错误消息中包含敏感信息，可能被攻击者利用


**审计要点**:

检查 catch 块中是否直接输出异常信息；检查是否泄露文件路径、数据库连接信息、堆栈跟踪


**修复建议**:

使用通用错误消息；记录详细日志仅供内部使用；使用自定义异常类隐藏敏感信息

### 问题分类GB/T34946-6.2.3.8 信息通过服务器日志文件泄露 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-532](https://cwe.mitre.org/data/definitions/532.html)


日志文件中记录敏感信息，可能被未授权访问


**审计要点**:

检查日志是否记录密码、密钥、token、session ID 等敏感信息；检查日志文件权限是否正确


**修复建议**:

过滤日志中的敏感信息；使用安全的日志框架和配置；确保日志文件权限正确

### 问题分类GB/T34946-6.2.3.9 信息通过调试日志文件泄露 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-532](https://cwe.mitre.org/data/definitions/532.html)


调试日志中包含敏感信息，可能被攻击者获取


**审计要点**:

检查 Debug.Write、Trace.Write、Console.Write 等调试输出是否包含敏感数据；检查调试日志是否在生产环境启用


**修复建议**:

生产环境关闭调试日志；避免记录敏感信息；使用条件编译控制调试代码

### 问题分类GB/T34946-6.2.3.10 信息通过持久Cookie泄露 🟠

**严重级别**: HIGH | **CWE**: [CWE-315](https://cwe.mitre.org/data/definitions/315.html)


在 Cookie 中存储敏感信息，可能被窃取或篡改


**审计要点**:

检查 Cookie 是否存储密码、密钥、信用卡号等敏感信息；检查是否设置适当的 HttpOnly 和 Secure 标志


**修复建议**:

避免在 Cookie 中存储敏感信息；使用 Session 存储；设置 HttpOnly、Secure、SameSite 属性

### 问题分类GB/T34946-6.2.3.11 未检查的输入作为循环条件 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-606](https://cwe.mitre.org/data/definitions/606.html)


使用未验证的用户输入作为循环条件，可能导致拒绝服务


**审计要点**:

检查循环条件（for、while 的终止条件）是否来自用户输入且未经范围验证


**修复建议**:

验证用户输入的范围；设置循环次数上限；使用安全的计数方式

### 问题分类GB/T34946-6.2.3.12 XPath注入 🟠

**严重级别**: HIGH | **CWE**: [CWE-91](https://cwe.mitre.org/data/definitions/91.html)


使用用户输入构造 XPath 查询，可能导致数据泄露或绕过认证


**审计要点**:

检查 XPathExpression、XPathNavigator 的输入是否来自用户且未验证；是否存在字符串拼接构建 XPath；是否使用 XPathVariableResolver 实现参数化查询


**修复建议**:

使用参数化 XPath 查询（XPathVariableResolver）；使用白名单验证输入；避免将用户输入直接用于 XPath

## 6.2.4 处理程序错误

### 问题分类GB/T34946-6.2.4.1 未限制危险类型文件的上传 🟠

**严重级别**: HIGH | **CWE**: [CWE-434](https://cwe.mitre.org/data/definitions/434.html)


允许上传可执行文件或危险类型文件，可能导致 RCE


**审计要点**:

检查文件上传功能是否验证文件类型；是否检查 MIME type 和文件扩展名；是否检查文件内容


**修复建议**:

使用白名单验证文件类型；检查文件头（魔数）；将上传文件重命名；存储在 Web 根目录外

## 6.2.5 不充分的封装

### 问题分类GB/T34946-6.2.5.1 违反信任边界 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-501](https://cwe.mitre.org/data/definitions/501.html)


在不可信区域使用内部数据，可能导致信息泄露


**审计要点**:

检查公共 API 是否暴露内部实现细节；检查敏感数据是否被传递到不受信任的区域


**修复建议**:

明确信任边界；在边界处验证和清理数据；避免暴露内部实现细节

## 6.2.6 安全功能

### 问题分类GB/T34946-6.2.6.1 明文存储口令 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-256](https://cwe.mitre.org/data/definitions/256.html)


在数据库或文件中以明文存储用户口令


**审计要点**:

检查口令存储是否使用明文；检查配置文件、数据库中口令字段是否为明文


**修复建议**:

使用强散列算法（bcrypt、scrypt、argon2）存储口令；使用盐值；避免硬编码口令

### 问题分类GB/T34946-6.2.6.2 存储可恢复的口令 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-257](https://cwe.mitre.org/data/definitions/257.html)


使用可逆加密存储口令，攻击者可能解密获取原始口令


**审计要点**:

检查口令存储是否使用可逆加密；检查是否使用加密而非散列


**修复建议**:

使用散列存储而非加密；使用强散列算法；避免使用可逆加密存储口令

### 问题分类GB/T34946-6.2.6.3 口令硬编码 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-798](https://cwe.mitre.org/data/definitions/798.html)


在源代码中硬编码口令或密钥


**审计要点**:

检查代码中是否存在硬编码的密码、API密钥、连接字符串；检查配置文件是否包含敏感信息


**修复建议**:

使用配置管理器或密钥管理服务存储敏感信息；环境变量；密钥管理系统（Azure Key Vault、AWS KMS）

### 问题分类GB/T34946-6.2.6.4 依赖Referer字段进行身份鉴别 🟠

**严重级别**: HIGH | **CWE**: [CWE-293](https://cwe.mitre.org/data/definitions/293.html)


仅依赖 HTTP Referer 头进行身份验证，容易被伪造


**审计要点**:

检查身份验证是否仅依赖 Request.UrlReferrer；是否存在基于 Referer 的访问控制


**修复建议**:

使用 Session、Token 等标准认证机制；Referer 仅作为辅助信息；使用 CSRF Token

### 问题分类GB/T34946-6.2.6.5 Cookie中的敏感信息明文存储 🟠

**严重级别**: HIGH | **CWE**: [CWE-315](https://cwe.mitre.org/data/definitions/315.html)


在 Cookie 中以明文存储敏感信息，可能被窃取或篡改


**审计要点**:

检查 Cookie 是否存储密码、密钥、信用卡号等敏感信息；是否设置 HttpOnly、Secure 属性；Cookie 内容是否加密或签名


**修复建议**:

敏感信息存储在服务器端 Session；设置 HttpOnly、Secure、SameSite 属性；如必须存储在 Cookie 中，应加密或签名

### 问题分类GB/T34946-6.2.6.6 敏感信息明文传输 🔴

**严重级别**: CRITICAL | **CWE**: [CWE-319](https://cwe.mitre.org/data/definitions/319.html)


在 HTTP 中以明文传输敏感信息，攻击者可通过中间人攻击截获敏感数据


**审计要点**:

检查登录、支付等敏感操作是否使用 HTTPS；检查是否跳过证书验证（ServicePointManager.ServerCertificateValidationCallback）


**修复建议**:

全站 HTTPS；使用 HSTS；正确配置 TLS 版本和密码套件；验证服务器证书

### 问题分类GB/T34946-6.2.6.7 使用已破解或危险的加密算法 🟠

**严重级别**: HIGH | **CWE**: [CWE-327](https://cwe.mitre.org/data/definitions/327.html)


使用不安全的加密算法如 DES、RC4


**审计要点**:

检查加密代码是否使用 DES、RC4、MD5、SHA1 等不安全算法；检查是否使用默认加密参数


**修复建议**:

使用 AES-256-GCM、ChaCha20 等安全算法；使用现代 TLS 配置；避免使用 ECB 模式

### 问题分类GB/T34946-6.2.6.8 可逆的散列算法 🟠

**严重级别**: HIGH | **CWE**: [CWE-328](https://cwe.mitre.org/data/definitions/328.html)


使用 MD5、SHA1 等弱散列算法存储口令，这些算法存在碰撞攻击风险


**审计要点**:

检查口令散列是否使用 MD5、SHA1、SHA256（不带盐）等弱算法；检查是否使用快速散列函数而非专门的密码哈希算法


**修复建议**:

使用 bcrypt、scrypt、argon2 等专门的口令散列算法；使用随机盐值；使用足够的工作因子

### 问题分类GB/T34946-6.2.6.9 密码分组链接模式未使用随机初始化矢量 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-329](https://cwe.mitre.org/data/definitions/329.html)


使用 CBC 模式加密时未使用随机 IV


**审计要点**:

检查对称加密是否使用固定的 IV；检查是否使用 CBC 模式且 IV 可预测


**修复建议**:

每次加密使用随机 IV；优先使用 GCM 等带认证的模式；使用 SecureRandom 生成 IV

### 问题分类GB/T34946-6.2.6.10 不充分的随机数 🟠

**严重级别**: HIGH | **CWE**: [CWE-338](https://cwe.mitre.org/data/definitions/338.html)


使用不安全的随机数生成器


**审计要点**:

检查是否使用 Random 类生成安全相关的随机数；检查是否使用 System.Random 代替 RNGCryptoServiceProvider


**修复建议**:

使用 RNGCryptoServiceProvider 或 RandomNumberGenerator 生成安全随机数；避免使用 Random

### 问题分类GB/T34946-6.2.6.11 安全关键的行为依赖反向域名解析 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-350](https://cwe.mitre.org/data/definitions/350.html)


依赖 DNS 反向解析进行安全决策，容易被欺骗


**审计要点**:

检查安全相关逻辑是否依赖 DNS 反向解析（如根据 IP 获取主机名进行信任判断）


**修复建议**:

避免依赖反向 DNS 进行安全决策；使用证书验证等更强的方式；记录并监控反向解析失败

### 问题分类GB/T34946-6.2.6.12 没有要求使用强口令 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-521](https://cwe.mitre.org/data/definitions/521.html)


未对口令复杂度进行要求


**审计要点**:

检查注册和修改口令时是否验证复杂度；是否检查长度、字符类型


**修复建议**:

设置最小长度要求（建议 8 位以上）；要求包含大小写字母、数字、特殊字符；检查常见弱口令

### 问题分类GB/T34946-6.2.6.13 没有对口令域进行掩饰 🟢

**严重级别**: LOW | **CWE**: [CWE-549](https://cwe.mitre.org/data/definitions/549.html)


登录表单中口令未掩码显示


**审计要点**:

检查密码输入框是否设置 PasswordChar 或 type=password


**修复建议**:

使用 type=password 或 PasswordChar；提供显示/隐藏密码的选项；避免明文显示口令

### 问题分类GB/T34946-6.2.6.14 依赖未经验证和完整性检查的Cookie 🟠

**严重级别**: HIGH | **CWE**: [CWE-565](https://cwe.mitre.org/data/definitions/565.html)


使用未验证的 Cookie 存储关键数据


**审计要点**:

检查 Cookie 内容是否经过签名或加密；检查是否存在 Cookie 篡改风险


**修复建议**:

对 Cookie 内容进行签名或加密；使用防篡改机制；验证 Cookie 完整性

### 问题分类GB/T34946-6.2.6.15 通过用户控制的SQL关键字绕过授权 🟠

**严重级别**: HIGH | **CWE**: [CWE-89](https://cwe.mitre.org/data/definitions/89.html)


使用用户输入构建 SQL 导致授权绕过，攻击者可访问未授权的数据


**审计要点**:

检查 SQL 查询中是否使用用户输入作为 ORDER BY、GROUP BY 列名；是否存在动态列名构建；MyBatis `${}` 是否用于表名/列名


**修复建议**:

使用白名单验证列名和表名；避免用户输入用于 SQL 关键字；使用参数化查询；对动态 SQL 元素做严格校验

### 问题分类GB/T34946-6.2.6.16 HTTPS会话中的敏感cookie没有设置安全属性 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-614](https://cwe.mitre.org/data/definitions/614.html)


HTTPS 响应的 Cookie 缺少安全标志


**审计要点**:

检查 Session Cookie、认证 Cookie 是否设置 Secure、HttpOnly 属性；检查是否在 HTTPS 下设置


**修复建议**:

设置 Cookie 的 Secure、HttpOnly、SameSite 属性；确保认证 Cookie 只在 HTTPS 传输

### 问题分类GB/T34946-6.2.6.17 未使用盐值计算散列值 🟠

**严重级别**: HIGH | **CWE**: [CWE-759](https://cwe.mitre.org/data/definitions/759.html)


散列计算未使用盐值，容易遭受彩虹表攻击


**审计要点**:

检查口令散列是否使用盐值；是否存在多个用户共用相同盐值的情况


**修复建议**:

每个用户使用随机生成的唯一盐值；使用专门的散列库（如 BCrypt）自动处理盐值

### 问题分类GB/T34946-6.2.6.18 RSA算法未使用最优非对称加密填充 🟠

**严重级别**: HIGH | **CWE**: [CWE-780](https://cwe.mitre.org/data/definitions/780.html)


使用不安全的 RSA 填充模式


**审计要点**:

检查 RSA 加密是否使用 PKCS1 v1.5 填充；检查是否存在自定义填充逻辑


**修复建议**:

使用 OAEP（PKCS1 v2）填充；避免使用 RSA 加密大量数据；使用混合加密方案

## 6.2.7 时间和方法状态

### 问题分类GB/T34946-6.2.7.1 会话固定 🟠

**严重级别**: HIGH | **CWE**: [CWE-384](https://cwe.mitre.org/data/definitions/384.html)


登录前后会话 ID 未改变，攻击者可劫持会话


**审计要点**:

检查用户登录后是否生成新的 Session ID；检查是否在认证时调用 Session.RegenerateId()


**修复建议**:

用户登录后调用 Session.RegenerateId()；避免在认证前使用 Session；设置合理的 Session 超时

### 问题分类GB/T34946-6.2.7.2 会话永不过期 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-613](https://cwe.mitre.org/data/definitions/613.html)


会话没有超时设置或超时时间过长


**审计要点**:

检查 Session 超时配置是否合理；检查是否在空闲一定时间后使 Session 失效


**修复建议**:

设置合理的 Session 超时时间（建议 15-30 分钟）；空闲超时；活动超时

## 6.2.8 Web问题

### 问题分类GB/T34946-6.2.8.1 跨站脚本（XSS） 🟠

**严重级别**: HIGH | **CWE**: [CWE-79](https://cwe.mitre.org/data/definitions/79.html)


未对输出进行转义导致 XSS 攻击


**审计要点**:

检查输出是否使用 Html.Encode 或 AntiXss 库；检查是否存在 Response.Write、@Model 等直接输出；检查 RichText 编辑器内容是否过滤


**修复建议**:

使用 Razor 的 @ 语法或 Html.Encode；对不受信任的数据进行 HTML/URL/JS 转义；使用 Content Security Policy

### 问题分类GB/T34946-6.2.8.2 跨站请求伪造（CSRF） 🟠

**严重级别**: HIGH | **CWE**: [CWE-352](https://cwe.mitre.org/data/definitions/352.html)


未实施 CSRF 防护，攻击者可诱导用户执行非预期操作


**审计要点**:

检查表单是否使用 @Html.AntiForgeryToken()；检查 AJAX 请求是否包含 CSRF Token；检查是否验证请求来源


**修复建议**:

使用 Anti-Forgery Token；验证 Request.Headers["Referer"] 或 Origin 头； SameSite Cookie

### 问题分类GB/T34946-6.2.8.3 HTTP响应拆分 🟠

**严重级别**: HIGH | **CWE**: [CWE-113](https://cwe.mitre.org/data/definitions/113.html)


未对用户输入进行验证导致 HTTP 响应拆分攻击


**审计要点**:

检查 Response.AddHeader、Response.Redirect 的输入是否包含 CR/LF 字符；检查是否存在原始字符串拼接


**修复建议**:

验证用户输入不包含 CR/LF（\r\n）；使用 API 而非字符串拼接；设置适当的响应头

### 问题分类GB/T34946-6.2.8.4 开放重定向 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-601](https://cwe.mitre.org/data/definitions/601.html)


使用用户输入控制重定向目标


**审计要点**:

检查 Response.Redirect、Response.RedirectToRoute 的 URL 参数是否来自用户输入且未验证


**修复建议**:

使用白名单验证重定向 URL；避免将用户输入直接用于重定向；使用相对路径

### 问题分类GB/T34946-6.2.8.5 依赖外部提供的文件的名称或扩展名 🟠

**严重级别**: HIGH | **CWE**: [CWE-434](https://cwe.mitre.org/data/definitions/434.html)


仅基于文件名或扩展名判断文件安全性，攻击者可通过伪造扩展名绕过检查


**审计要点**:

检查文件上传功能是否仅检查文件扩展名而非文件内容（魔数）；是否存在根据文件名执行不同逻辑的情况；是否防止了双扩展名绕过（如 .jpg.php）


**修复建议**:

检查文件内容（魔数/文件头）而非仅依赖扩展名；使用白名单验证允许的文件类型；将上传文件重命名为随机名称；存储在 Web 根目录外

## 6.2.9 用户界面错误

### 问题分类GB/T34946-6.2.9.1 点击劫持 🟡

**严重级别**: MEDIUM | **CWE**: [CWE-1021](https://cwe.mitre.org/data/definitions/1021.html)


页面未设置防护，攻击者可将页面嵌入 iframe 进行点击劫持


**审计要点**:

检查响应头是否设置 X-Frame-Options；检查 Content-Security-Policy 是否设置 frame-ancestors


**修复建议**:

设置 X-Frame-Options: DENY 或 SAMEORIGIN；使用 Content-Security-Policy: frame-ancestors 'self'

## 误报排除指南


以下情况通常为误报，不需要修复：


- **工具类/工具函数**：DateUtil、StringUtil、JSONUtil 等通用工具类的方法本身不是漏洞
- **测试代码**：单元测试、集成测试中的代码通常不需要修复
- **示例代码**：Demo、Example 等示例代码
- **配置类**：Properties/yml 配置加载通常不涉及用户输入
- **常量定义**：final static 常量是安全的
- **白名单验证**：已有白名单校验的代码片段
