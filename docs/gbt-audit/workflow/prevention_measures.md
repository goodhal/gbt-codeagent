# 审计覆盖率预防措施

## 问题回顾

本次审计中，LLM审计仅覆盖了 **35.4%** 的源代码文件（45/127），导致5个预埋漏洞未被检出。这违反了SKILL.md中的强制要求：**LLM审计必须覆盖项目中的全部源代码文件**。

## 预防措施体系

### 一、流程预防（审计前）

#### 1.1 强制文件计数检查

**执行时机**：LLM审计开始前

**操作步骤**：
```bash
# 步骤1：获取全部源代码文件列表
Glob path="E:\code\JavaSecLab\src\main\java" pattern="**/*.java"

# 步骤2：记录文件总数
总文件数 = 127

# 步骤3：确认审计策略
审计策略 = "全部文件审计"
```

**检查清单**：
- [ ] 已使用Glob获取全部代码文件
- [ ] 已记录文件总数
- [ ] 已确认审计策略为"全部文件审计"
- [ ] 已理解"不得遗漏任何文件"的要求

**输出确认格式**：
```
【审计前文件计数确认】
- 源代码文件总数: 127
- 审计策略: 全部文件审计
- 覆盖率要求: 100%
- 下一步: 开始LLM审计
```

### 二、执行预防（审计中）

**执行时机**：每完成一个批次的审计

**跟踪表格**：
```

| 批次 | 计划文件数 | 实际审计数 | 覆盖率 | 状态 | 备注 |

| 4 | 25 | 0 | 35.4% | ❌ 未开始 | |
| 5 | 27 | 0 | 35.4% | ❌ 未开始 | |
```

**检查点**：
- 每完成一个批次，立即更新进度表格
- 如果实际审计数 < 计划文件数，立即调查原因
- 覆盖率低于预期时，暂停审计，查明原因

#### 2.2 中期覆盖率检查

**执行时机**：审计进行到50%时

**检查命令**：
```bash
python scripts/audit_coverage_checker.py \
  --target "E:\code\JavaSecLab\src\main\java" \
  --llm_audit "findings/llm_audit" \
  --baseline "findings/baseline"
```

**预期结果**：
- 覆盖率应 ≥ 50%
- 如果 < 50%，说明审计进度落后，需要加快或增加资源

### 三、验证预防（审计后）

#### 3.1 强制覆盖率检查

**执行时机**：LLM审计完成后，生成报告前

**检查命令**：
```bash
python scripts/audit_coverage_checker.py \
  --target "E:\code\JavaSecLab\src\main\java" \
  --llm_audit "findings/llm_audit" \
  --baseline "findings/baseline" \
  --output "coverage_check_report.md"
```

**通过标准**：
- ✅ LLM审计覆盖率 = 100%
- ⚠️ LLM审计覆盖率 ≥ 95%（需说明原因）
- ❌ LLM审计覆盖率 < 95%（必须补充审计）

**未通过处理流程**：
```
覆盖率检查不通过
    ↓
获取未审计文件列表
    ↓
补充审计未审计文件
    ↓
重新运行覆盖率检查
    ↓
覆盖率达标 → 继续生成报告
覆盖率不达标 → 继续补充审计
```

#### 3.2 审计完整性验证

**验证内容**：
1. **文件覆盖验证**：每个源代码文件是否都被审计
2. **漏洞类型验证**：每类漏洞是否都有审计记录
3. **Controller验证**：每个Controller是否都被审计

**验证命令**：
```bash
# 验证Controller覆盖
python scripts/audit_coverage_checker.py \
  --target "E:\code\JavaSecLab\src\main\java" \
  --llm_audit "findings/llm_audit" \
  --baseline "findings/baseline" \
  --check-controllers
```

### 四、自动化预防（工具支持）

#### 4.1 覆盖率检查工具

**工具位置**：`scripts/audit_coverage_checker.py`

**功能**：
- 自动扫描源代码文件
- 自动扫描LLM审计文件
- 计算覆盖率
- 生成未审计文件列表
- 生成检查报告

**使用方法**：
```bash
# 基本用法
python scripts/audit_coverage_checker.py \
  --target <项目目录> \
  --llm_audit <llm_audit目录> \
  --baseline <baseline目录>

# 生成报告
python scripts/audit_coverage_checker.py \
  --target <项目目录> \
  --llm_audit <llm_audit目录> \
  --baseline <baseline目录> \
  --output <报告输出路径>
```

**退出码**：
- 0：覆盖率达标（100%）
- 1：覆盖率不达标（< 100%）

#### 4.2 审计进度跟踪脚本

### 步骤 5c：审计覆盖率验证（强制）

**执行命令**：

**通过标准**：
- ✅ 覆盖率 = 100%：继续步骤 6

**违反后果**：
- 覆盖率不达标时生成报告，可能导致遗漏关键漏洞
- 检出率下降，影响审计质量
```

### 五、文化预防（团队规范）

#### 5.1 审计原则宣导

**核心原则**：
- **全面覆盖**：LLM审计必须覆盖全部源代码文件
- **不得遗漏**：不因文件类型、位置、重要性而跳过
- **质量优先**：宁可慢，不可漏

**宣导方式**：
- 在SKILL.md中加粗强调
- 在audit_workflow.md中详细说明
- 在审计开始前输出确认信息

#### 5.2 审计质量文化

**质量指标**：
- 文件覆盖率：100%（强制）
- 漏洞检出率：>95%（目标）
- 误报率：<5%（目标）

**质量承诺**：
- 不达到100%覆盖率，不生成审计报告
- 发现遗漏，立即补充审计
- 定期回顾审计质量，持续改进
